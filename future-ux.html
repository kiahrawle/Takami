Automated Personalization

The 5-Layer
Personalization Engine

Core principle: automate the thinking, not the ritual. Users want clinic-level precision without clinic-level time investment. The system becomes their digital dermatologist.

1
Skin Truth Baseline — One-Time Setup
+
Clinic/device scan → AI extracts 12 key metrics (turnover rate, barrier integrity, sebum balance, sensitivity index, hydration level, pore visibility, clarity score, texture grade, UV damage estimate, pigmentation map, elasticity, pH balance)
User confirms or edits → Creates a "golden profile" — the permanent single source of truth
Stored across all channels: app, website, in-clinic device, partner platforms
No repeat questionnaires — this is the foundation everything builds from
2
Daily Context Engine — Zero-Input Updates
+
Weather / season → Auto-adjusts hydration vs. peel intensity recommendations
Usage detection → Tracks actual application via app reminders and reorder patterns
Life events → Pregnancy, travel, stress flags trigger automatic gentle routine shifts
Output: "Today: +10% hydration recommended. Peel intensity: hold steady."
3
Predictive Routine Builder — Smart Defaults
+
Hero product first — always starts with Soft Peel at optimal frequency for current state
2–3 complementary products maximum — never suggests "add everything"
Visual sequence → Drag-to-reorder with AI explaining conflicts ("Don't layer Vitamin C before Peel")
One-tap approve → 90% acceptance rate target. Decision fatigue is the enemy.
4
Progress Tracking — Motivational Feedback
+
Weekly delta → "Roughness −12%, Clarity +8% since last month" — specific, quantified, personal
Visual skin journey → Before/after heatmaps using anonymized aggregate clinic data for comparison
Gentle nudges → "Your barrier strengthened; Peel frequency can safely increase to daily"
Wabi-sabi framing: visible improvement over perfection promises builds authentic, lasting trust
5
Human-in-the-Loop Safety — Trust Anchor
+
Threshold alerts → "Unusual dryness detected; consult clinic?" — AI defers to human when uncertain
Clinic sync → Dermatologist can override AI suggestions in real-time via secure dashboard
Undo history → "Revert to Feb 15 routine" — 30-day buffer, 1-tap revert, zero judgment
Clinic remains final authority — automation earns trust by knowing its own limits

Key UX Strategies

Trust-first
automation design

Strategy 01

Progressive Trust Building

Phase 1: raw metrics + clinic correlation. Phase 2: suggest routine with "why we chose this." Phase 3: track and prove — "your clarity improved exactly as predicted."

Strategy 02

Behavioral Guardrails (Japanese-Tuned)

✓ "Most clinic patients in your condition start here"  ✓ "Undo available anytime via Clinic Sync"  ✗ No "BUY NOW" pressure  ✗ Never more than 3 steps visible

Strategy 03

Safety Architecture

Every prediction cites its source. Human override via clinic chat (2-hour SLA). 30-day routine history with 1-tap revert. "Feels wrong?" → Immediate human handoff. Automation earns trust by knowing its limits.
